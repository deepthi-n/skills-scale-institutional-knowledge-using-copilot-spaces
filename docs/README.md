# OctoAcme Project Management Docs

Welcome! This README centralizes knowledge about OctoAcme's project management practices. It summarizes our cross-functional, iterative approach and provides links to all official process documents to facilitate onboarding, ongoing collaboration, quality, and consistent delivery.

---

## Project Management Processes Summary

OctoAcme's project management process is guided by these core principles:
- **Customer-first focus** – Deliver solutions that prioritize customer value and usability.
- **Iterative, agile development** – Work in small, testable increments, enabling rapid feedback and continuous learning.
- **Defined ownership and roles** – Every project is supported by a Project Manager (PM) and a Product Manager (PdM), with defined responsibilities for developers, QA, and stakeholders.
- **Collaborative execution and structured communication** – Regular standups, PM/PdM alignment, and transparent stakeholder reporting keep everyone informed and risks visible.
- **Continuous improvement** – Retrospectives, risk management, and lessons-learned sessions fuel process refinements and quality assurance.

The typical workflow consists of five main phases: **Initiation** (problem and stakeholders), **Planning** (scope, resources, milestones), **Execution** (implementation, testing, review), **Release** (deployment, verification), and **Close & Retrospective** (learning and improvements). Key communication moments include weekly PM/PdM syncs, biweekly team standups, monthly stakeholder updates, and ad-hoc escalation. Quality is ensured through code reviews, automated and manual testing, and living documentation of process artifacts. All work is tracked in GitHub Projects, and improvement opportunities are continuously fed back into this documentation.
